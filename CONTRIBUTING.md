# CONTRIBUTING

Thank you for considering contributing to react-native-leaflet! Here are some guidelines.

## Contribution Process

1. **Fork** the repository
2. **Clone** it to your local machine
3. **Create a branch** for your feature: `git checkout -b feature/AmazingFeature`
4. **Make the necessary changes**
5. **Test** your changes
6. **Commit**: `git commit -m 'Add some AmazingFeature'`
7. **Push** to the branch: `git push origin feature/AmazingFeature`
8. **Open a Pull Request**

## Guidelines

### Code

- Use TypeScript for all main files
- Keep types well defined
- Add comments to complex functions
- Follow the existing code pattern

### Commit Messages

- Use the conventional standard:
  - `feat: add new feature`
  - `fix: fix bug`
  - `docs: update documentation`
  - `test: add tests`
  - `refactor: refactor code`

### Pull Requests

- Clearly describe what was changed
- Reference related issues: `Closes #123`
- Provide examples if applicable
- Update documentation

### Issues

- Check if the issue doesn't already exist
- Be specific in the description
- Provide steps to reproduce (bugs)
- Include code examples if possible

## Local Development

```bash
# Install dependencies
npm install

# Build
npm run build

# Development with watch
npm run dev

# Run tests
npm run test

# Lint
npm run lint
```

## Report Bugs

When reporting a bug, include:

- React and Leaflet versions
- Browser and version
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots (if applicable)

## Suggest Improvements

When suggesting improvements:

- Explain the use case
- Provide code examples
- Explain how this would improve the library
